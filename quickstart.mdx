This guide takes you from a clone to a running effy agent you can talk to. Locally, you don't need to mint tokens — a dev-auth flag stands in for your management layer.

## Prerequisites

- **Node.js 24** (the project pins `"node": "24.x"`).
- The [effy repository](https://github.com/agentico-labs/effy).
- Model gateway credentials. effy routes the model through the Vercel AI Gateway; see the [eve deployment docs](https://eve.dev/docs) for which variables to set.

## Get started

<Steps>
  <Step title="Install">
    Clone the repository and install dependencies.

    ```bash
    git clone https://github.com/agentico-labs/effy.git
    cd effy
    npm install
    ```
  </Step>
  <Step title="Run the dev REPL">
    Start `eve dev` with dev auth enabled. This identifies you as tenant `dev`, user `dev-user`, so memory and tools have a scope to attach to without a token.

    ```bash
    EFFY_DEV_AUTH=1 npm run dev
    ```

    The REPL opens an interactive session. Try asking it to remember something:

    ```text
    > Remember that I prefer metric units.
    ```

    The agent calls its `remember` tool and confirms. Ask a follow-up and it recalls the fact — long-term memory is injected on every turn.
  </Step>
  <Step title="Drive the HTTP API (optional)">
    `eve dev` also serves effy's HTTP API. Note the port it prints (default `3000`). With `EFFY_DEV_AUTH=1`, the `x-effy-*` headers stand in for a signed token:

    ```bash
    curl -X POST http://127.0.0.1:3000/eve/v1/session \
      -H 'content-type: application/json' \
      -H 'x-effy-tenant: acme' \
      -H 'x-effy-user: alice' \
      -H 'x-effy-profile: {"name":"Ada","personality":"Warm and concise."}' \
      -d '{"message":"Remember my favorite color is teal."}'
    ```

    The `x-effy-profile` header sets a [persona](/concepts/personas) inline for this user. The response returns a `sessionId` and a `continuationToken` you reuse for follow-ups.
  </Step>
</Steps>

<Warning>
  `x-effy-tenant`, `x-effy-user`, and `x-effy-profile` are honored **only** when `EFFY_DEV_AUTH=1`. Never set that variable in production — it would let any caller claim any identity. In production, callers authenticate with a [signed session token](/guides/authentication).
</Warning>

## Useful scripts

```bash
npm run dev         # eve dev REPL (set EFFY_DEV_AUTH=1 for local auth)
npm run typecheck   # tsc
npm test            # node --test (auth self-check)
npm run build       # eve build
```
